# Define Navidrome/OpenSubsonic Scope

Research date: 2026-07-07

## Recommendation

TMU should implement a **read-only, queue-first Navidrome/OpenSubsonic client** for the MVP: token-authenticated JSON API calls, `ping` validation, playlist browsing, ID3 artist/album browsing, song search, authenticated `stream` URLs, optional `getCoverArt` image retrieval, and `scrobble` reporting for now-playing and completed plays. TMU should not implement playlist writes, ratings/stars, shares, bookmarks, server-side play queues, media-library scan controls, internet radio, user management, folder browsing, video, lyrics, or OpenSubsonic-only extensions in the MVP; the selected endpoint set is enough to enqueue Navidrome tracks into the shared queue while matching the already-decided local-file, YouTube-download-only, Bun/TypeScript, and mpv playback scope supplied with this ticket. Sources: [OpenSubsonic API method groups](https://raw.githubusercontent.com/opensubsonic/open-subsonic-api/main/content/en/docs/opensubsonic-api.md), [Navidrome supported endpoints and limitations](https://raw.githubusercontent.com/navidrome/website/master/content/en/docs/developers/subsonic-api.md), [cliamp Navidrome client capability checks](/home/txchen/code/github/cliamp/external/navidrome/client.go:25).

Use **Subsonic token+salt auth by default**, not a separate login session. Every request should include `u`, `t=md5(password + salt)`, `s`, `v`, `c`, and `f=json`; the API reference says either `p` or both `t` and `s` must be present, recommends token auth starting with API 1.13.0, and defines `f=json` as the JSON response selector. OpenSubsonic API-key auth exists, but it is an extension and conflicts with `u`/`p`/`t`/`s`, so TMU should postpone API-key auth unless the setup flow later detects and validates it explicitly. Sources: [OpenSubsonic common parameters and authentication](https://raw.githubusercontent.com/opensubsonic/open-subsonic-api/main/content/en/docs/api-reference.md), [cliamp token URL builder](/home/txchen/code/github/cliamp/external/navidrome/client.go:182).

Set `v=1.16.1` for the Navidrome MVP target and treat version/auth errors as setup failures. Navidrome documents compatibility with Subsonic API v1.16.1 with exceptions, while the selected ID3 browsing endpoints start at API 1.8.0, raw stream format starts at API 1.9.0, and token authentication starts at API 1.13.0. Sources: [Navidrome Subsonic API compatibility](https://raw.githubusercontent.com/navidrome/website/master/content/en/docs/developers/subsonic-api.md), [getArtists docs](https://raw.githubusercontent.com/opensubsonic/open-subsonic-api/main/content/en/docs/Endpoints/getartists.md), [stream docs](https://raw.githubusercontent.com/opensubsonic/open-subsonic-api/main/content/en/docs/Endpoints/stream.md), [OpenSubsonic auth docs](https://raw.githubusercontent.com/opensubsonic/open-subsonic-api/main/content/en/docs/api-reference.md).

## Data Mapping

TMU's Navidrome track model should keep IDs as strings, carry title, artist, album, artist ID, album ID, year, track number, genre, duration, cover-art ID, content type/suffix if present, and provider metadata containing the Navidrome/Subsonic song ID; Navidrome explicitly warns that its IDs are strings, and OpenSubsonic response examples expose these song and album fields. Sources: [Navidrome ID warning](https://raw.githubusercontent.com/navidrome/website/master/content/en/docs/developers/subsonic-api.md), [getAlbum song fields](https://raw.githubusercontent.com/opensubsonic/open-subsonic-api/main/content/en/docs/Endpoints/getalbum.md), [getPlaylist entry fields](https://raw.githubusercontent.com/opensubsonic/open-subsonic-api/main/content/en/docs/Endpoints/getplaylist.md), [cliamp `subsonicSong` mapping](/home/txchen/code/github/cliamp/external/navidrome/client.go:453).

TMU should not copy cliamp's current omission of `coverArt` in its Navidrome mapping if the MVP UI renders artwork, because OpenSubsonic examples include `coverArt` on artists, albums, and songs, while cliamp's local `subsonicSong` and `subsonicAlbum` structs only map IDs, display metadata, counts, genre, and duration. Sources: [getArtists coverArt examples](https://raw.githubusercontent.com/opensubsonic/open-subsonic-api/main/content/en/docs/Endpoints/getartists.md), [getAlbum coverArt examples](https://raw.githubusercontent.com/opensubsonic/open-subsonic-api/main/content/en/docs/Endpoints/getalbum.md), [cliamp song/album structs](/home/txchen/code/github/cliamp/external/navidrome/client.go:453).

Build stream URLs at play/enqueue time rather than persisting them as durable queue identity, because the URL embeds request authentication material (`u`, `t`, `s`) and the stable identity is the server URL plus song ID. The API reference defines those auth parameters as request parameters, and cliamp uses the stream URL as the playable `Path` while storing the song ID separately in provider metadata. Sources: [OpenSubsonic common parameters](https://raw.githubusercontent.com/opensubsonic/open-subsonic-api/main/content/en/docs/api-reference.md), [cliamp `songToTrack`](/home/txchen/code/github/cliamp/external/navidrome/client.go:466).

## Pagination And Loading Policy

Only `getAlbumList2` and `search3` are server-paginated in the MVP endpoint set. `getAlbumList2` has `size` with max 500 and `offset`, while `search3` has separate count/offset pairs for artists, albums, and songs. Sources: [getAlbumList2 pagination parameters](https://raw.githubusercontent.com/opensubsonic/open-subsonic-api/main/content/en/docs/Endpoints/getalbumlist2.md), [search3 paging parameters](https://raw.githubusercontent.com/opensubsonic/open-subsonic-api/main/content/en/docs/Endpoints/search3.md).

Use page sizes around 100 for album list and search in the TUI, and request the next page only when the user scrolls near the end. This recommendation stays below the documented `getAlbumList2` max of 500, avoids loading a large remote catalog up front, and matches the queue-first MVP constraint. Sources: [getAlbumList2 max size](https://raw.githubusercontent.com/opensubsonic/open-subsonic-api/main/content/en/docs/Endpoints/getalbumlist2.md), [search3 paging parameters](https://raw.githubusercontent.com/opensubsonic/open-subsonic-api/main/content/en/docs/Endpoints/search3.md).

Load `getArtists` as one async request and cache it in memory for the session, because `getArtists` has no documented `size` or `offset` parameter and returns artists grouped inside indexes. Sources: [getArtists parameters/result](https://raw.githubusercontent.com/opensubsonic/open-subsonic-api/main/content/en/docs/Endpoints/getartists.md), [cliamp artist flattening](/home/txchen/code/github/cliamp/external/navidrome/client.go:323).

Do not mirror Navidrome's library into a local database in the MVP. Use short-lived in-memory caches for playlists, playlist tracks, artists, and selected album pages, with an explicit refresh action; cliamp caches playlists and playlist tracks and exposes `Refresh` to clear those caches. Sources: [cliamp playlist cache](/home/txchen/code/github/cliamp/external/navidrome/client.go:238), [cliamp track cache and refresh](/home/txchen/code/github/cliamp/external/navidrome/client.go:278).

## Reporting Behavior

TMU must not assume that streaming a song updates play counts. OpenSubsonic says servers must not count `stream` access as a play and clients can use `scrobble`; Navidrome separately documents that it does not mark songs played on `stream` and only does so when `scrobble` is called with `submission=true`. Sources: [OpenSubsonic stream note](https://raw.githubusercontent.com/opensubsonic/open-subsonic-api/main/content/en/docs/Endpoints/stream.md), [Navidrome stream/scrobble note](https://raw.githubusercontent.com/navidrome/website/master/content/en/docs/developers/subsonic-api.md).

TMU should send `submission=false` once per started Navidrome track to update server-side now-playing state, then send `submission=true` with `time=<Date.now()>` only after the track is meaningfully played. The Subsonic/OpenSubsonic `scrobble` endpoint defines `submission` as the distinction between submission and now-playing notification and defines `time` as milliseconds since 1970; cliamp uses `submission=false` for now-playing, `submission=true` for completed reports, and a 50% playback threshold before reporting completion. Sources: [scrobble parameters](https://raw.githubusercontent.com/opensubsonic/open-subsonic-api/main/content/en/docs/Endpoints/scrobble.md), [cliamp scrobble call construction](/home/txchen/code/github/cliamp/external/navidrome/client.go:525), [cliamp 50% threshold](/home/txchen/code/github/cliamp/ui/model/notifications.go:121).

Expose a config flag equivalent to `navidrome.scrobble = false` and default it to enabled. cliamp's config treats `scrobble = false` as the explicit opt-out and only reports playback when the track has Navidrome metadata and scrobbling is not disabled. Sources: [cliamp Navidrome config fields](/home/txchen/code/github/cliamp/config/config.go:66), [cliamp scrobble config parse](/home/txchen/code/github/cliamp/config/config.go:367), [cliamp `CanReportPlayback`](/home/txchen/code/github/cliamp/external/navidrome/client.go:513).

Reporting errors should be logged or surfaced as low-priority diagnostics, not playback failures. The API call is annotation rather than media retrieval, and cliamp dispatches now-playing/scrobble reports asynchronously and ignores network failures inside the Navidrome client. Sources: [scrobble docs](https://raw.githubusercontent.com/opensubsonic/open-subsonic-api/main/content/en/docs/Endpoints/scrobble.md), [cliamp async now-playing/scrobble dispatch](/home/txchen/code/github/cliamp/ui/model/notifications.go:107), [cliamp best-effort scrobble](/home/txchen/code/github/cliamp/external/navidrome/client.go:525).

## Compatibility Risks

**Version and auth negotiation:** token auth requires API 1.13.0, API-key auth is OpenSubsonic-specific, and servers can return specific errors for unsupported token/API-key/conflicting authentication mechanisms. TMU should validate with `ping`, show the returned API error message, and avoid sending both API-key and token/password parameters in one request. Sources: [OpenSubsonic auth docs](https://raw.githubusercontent.com/opensubsonic/open-subsonic-api/main/content/en/docs/api-reference.md), [OpenSubsonic error codes](https://raw.githubusercontent.com/opensubsonic/open-subsonic-api/main/content/en/docs/api-reference.md).

**Non-Navidrome endpoint behavior:** the MVP endpoint set is standard Subsonic/OpenSubsonic, but TMU is explicitly targeting Navidrome semantics first. Navidrome documents exceptions such as no video support, simulated browse-by-folder, no `getPlaylists username`, no Lucene queries for `search2`/`search3`, and string IDs; TMU avoids those risks by using ID3 browsing, omitting `username`, using simple search queries, and keeping IDs as strings. Source: [Navidrome compatibility notes](https://raw.githubusercontent.com/navidrome/website/master/content/en/docs/developers/subsonic-api.md).

**Folder-browsing incompatibility:** Navidrome has no plans to support real browse-by-folder and simulates `getIndexes`/`getMusicDirectory`, so TMU should not implement the Navidrome MVP around folder endpoints. OpenSubsonic's own guidance says ID3-tag browsing uses `getArtists`, `getArtist`, and `getAlbum`, while file-structure browsing uses `getIndexes` and `getMusicDirectory`. Sources: [Navidrome folder-browsing note](https://raw.githubusercontent.com/navidrome/website/master/content/en/docs/developers/subsonic-api.md), [OpenSubsonic ID3-vs-folder guidance](https://raw.githubusercontent.com/opensubsonic/open-subsonic-api/main/content/en/docs/opensubsonic-api.md).

**Search semantics:** `search3` supports paging, but Navidrome says it does not support Lucene queries and only supports simple autocomplete queries. TMU should send plain user text and not expose syntax that depends on Lucene or server-specific search parsing. Sources: [search3 docs](https://raw.githubusercontent.com/opensubsonic/open-subsonic-api/main/content/en/docs/Endpoints/search3.md), [Navidrome search note](https://raw.githubusercontent.com/navidrome/website/master/content/en/docs/developers/subsonic-api.md).

**Cover-art IDs:** original Subsonic allowed `getCoverArt` IDs to refer to several entity types, while OpenSubsonic clarifies that `id` refers to a cover-art ID only. TMU should therefore use the `coverArt` field returned by browse/search responses rather than substituting artist, album, or song IDs. Source: [getCoverArt docs](https://raw.githubusercontent.com/opensubsonic/open-subsonic-api/main/content/en/docs/Endpoints/getcoverart.md).

**Stream play-count behavior:** because OpenSubsonic and Navidrome both require `scrobble` for play counts, compatibility with older or non-OpenSubsonic servers should be measured by whether `scrobble` works, not by whether `stream` side effects happen. Sources: [OpenSubsonic stream note](https://raw.githubusercontent.com/opensubsonic/open-subsonic-api/main/content/en/docs/Endpoints/stream.md), [Navidrome stream/scrobble note](https://raw.githubusercontent.com/navidrome/website/master/content/en/docs/developers/subsonic-api.md).

**Raw streams and seeking:** `format=raw` is the best MVP default because it disables transcoding and gives mpv the original media, but `timeOffset` for music is only required when a server supports the OpenSubsonic Transcode Offset extension. TMU should let mpv seek normally against the stream URL and should not build MVP seek behavior around `timeOffset`. Source: [stream parameters](https://raw.githubusercontent.com/opensubsonic/open-subsonic-api/main/content/en/docs/Endpoints/stream.md).

## Decision

Implement this Navidrome provider surface for TMU MVP:

1. `ping` during setup and on manual reconnect.
2. `getPlaylists` and `getPlaylist` for read-only playlist browsing.
3. `getArtists`, `getArtist`, `getAlbumList2`, and `getAlbum` for full Navidrome artist/album browsing.
4. `search3` as song search with pagination and queue enqueue actions.
5. `stream?id=<songId>&format=raw` URLs generated with token auth and handed to mpv.
6. `getCoverArt` using returned `coverArt` IDs where the UI needs images.
7. `scrobble` with `submission=false` on playback start and `submission=true&time=<unixMillis>` after the local scrobble threshold, with a user opt-out.

Postpone playlist writes, folder browsing, stars/ratings, lyrics, downloads, OpenSubsonic extension negotiation, API-key auth, library sync/offline mirror, and non-Navidrome compatibility work beyond this standard endpoint subset.
